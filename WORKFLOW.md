# Project Workflow

## Purpose

This project automatically collects current large language model information from selected vendor sources and generates a structured comparison report for client reference and model selection.

---

# High-Level Workflow

```text
Official Links
        ↓
Claude Code Command
        ↓
Researcher Agent
        ↓
Structured Draft Output
        ↓
Reporter Agent
        ↓
Final Report (models.md)
```

---

# Core Components

| Component | Responsibility |
|---|---|
| `links.md` | Official source list |
| `.claude/commands/` | User-facing workflow commands |
| `BaseResearcher.md` | Shared research rules |
| `Researcher-*.md` | Vendor-specific Researchers |
| `Researcher-Vendor.md` | Generic Researcher for additional vendors |
| `drafts/*.md` | Structured intermediate outputs |
| `Reporter.md` | Final synthesis and recommendations |
| `models.md` | Final client-facing report |

---

# Research Workflow

Each Researcher:

```text
Read links.md
        ↓
Use official sources first
        ↓
Extract structured model information
        ↓
Generate drafts/<vendor>.md
```

Researcher rules:

- prioritise official sources
- avoid unsupported claims
- standardise pricing and context formatting
- distinguish API vs open-weight models
- preserve source traceability

---

# Reporting Workflow

The Reporter:

```text
Read drafts/*.md
        ↓
Compare vendors
        ↓
Generate recommendations
        ↓
Write models.md
```

The Reporter does not browse the web.

---

# Supported Workflows

## Update One Core Vendor

```text
/track-openai
/generate-report
```

---

## Update an Additional Vendor

```text
/track-vendor Mistral
/generate-report
```

Additional vendors are defined through `links.md`.

---

## Refresh All Drafts

```text
/track-all
```

This updates:

- core vendors
- additional vendors discovered in `links.md`

---

## Full Refresh

```text
/update-report
```

This refreshes all drafts and rebuilds `models.md`.

---

# Draft File Role

Files inside `drafts/` are intermediate structured outputs.

They are generated by Researcher agents and read by the Reporter.

They are not the final deliverable.

---

# Recommendation Philosophy

The system uses AI-assisted reasoning instead of hard-coded rankings.

Recommendations consider:

- pricing
- context length
- modality support
- deployment suitability
- practical strengths

---

# Source Policy

Preferred sources:

- official documentation
- official pricing pages
- official GitHub repositories
- official Hugging Face organisation pages
- official model cards

Unofficial blogs and random comparison websites should not be treated as authoritative sources.

---

# Output Goal

The goal is not to create a benchmark leaderboard.

The goal is to help answer:

```text
Which model should we consider for this situation, and why?
```
